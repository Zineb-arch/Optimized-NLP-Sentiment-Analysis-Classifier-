# Model 3: GRU with Pretrained Gensim Word2Vec & Hyperparameter Tuning

## Model Architecture
This is the most advanced model of the three, leveraging transfer learning via custom-trained embeddings.
1. **Embedding Layer (Pretrained):** Initialized with a matrix generated by `Gensim Word2Vec`. Instead of starting randomly, the model already understands semantic similarities between words.
2. **GRU Layer:** 128 units. A Gated Recurrent Unit is slightly more computationally efficient than an LSTM while providing similar abilities to remember long-term context.
3. **Dense Layers:** 64 units (ReLU) followed by a Dropout layer (0.5).
4. **Output Layer:** 1 unit (Sigmoid).

## Techniques Applied
* **Pretrained Word Embeddings:** Used the `gensim` library to train a Word2Vec model on the training corpus. Extracted the weights into an embedding matrix injected into the Keras layer.
* **Hyperparameter Tuning:** 
  * Adjusted the GRU layer units to 128.
  * Increased batch size to 128.
  * Defined a custom `Adam` optimizer with a tuned initial learning rate of `0.001`.
* **Dynamic Learning Rate (Optimization):** Implemented the `ReduceLROnPlateau` callback to automatically cut the learning rate in half if the validation loss stagnates, allowing the model to converge more accurately in later epochs.
* **Early Stopping & Dropout:** Combined recurrent dropout (0.3) and spatial dropout (0.5) to heavily regularize the complex network.
* **Pre-padding for Sequence Optimization:** Applied `padding='pre'` instead of post-padding. Because Recurrent Neural Networks process data sequentially, placing the zeros at the beginning ensures the model processes the actual words immediately before making a prediction, rather than reading the words and then forgetting the context after processing hundreds of trailing zeros.
